Fields

Puggle supports fielded data. When performing a search you can either specify a field, or use the default field. The field names and default field is implementation specific.

You can search any field by typing the field name followed by a colon ":" and then the term you are looking for.

The available fields in Puggle are the following: 

  • app:
    The query [ app:term ] restricts results to applications containing term in the file's name. For instance, [ intitle:calc ] will return applications that mention the word notepad in their names, like the C:\WINDOWS\system32\calc.exe. 
  • filename:
    The query filename:term restricts results to documents containing term in the file's name. For instance, [ flu shot intitle:help ] will return documents that mention the word help in their names, and mention the words flu and shot anywhere in the document. 
  • filetype:
    If you include filetype:suffix in your query, Puggle will restrict the results to pages whose names end in suffix. For example, [ web page evaluation checklist filetype:pdf ] will return Adobe Acrobat pdf files that match the terms web, page, evaluation, and checklist. You can restrict the results to pages whose names end with pdf and doc by using the OR operator, e.g. [ email security filetype:pdf OR filetype:doc ]. 
  • title:
    The query [ title:term ] restricts results to documents containing term in the title. For instance, [ flu shot title:help ] will return documents that mention the word help in their titles, and mention the words flu and shot anywhere in the document (title or not). 
  • inpath:
    If you include inpath: in your query, Puggle will restrict the results to files containing that word in the path. For instance, [ inpath:documents ] searches for files in which their full path contains the word print. The query [ inurl:healthy eating ] will return files that mention the words healthy in their path, and mention the word eating anywhere in the document.

Note: The field is only valid for the term that it directly precedes, so the query

inpath:Do it right

Will only find Do in the inpath field. It will find it and right in the content of the file.
